## Context and Problem

After classification, Phase 3 decides what remediation action to take per label per confidence score. The spec defines a threshold-gated decision matrix:

| Label + confidence | Action |
|---|---|
| `charm-bug` ≥ threshold | Generate unified diff patch → `fix.patch` |
| `test-bug` ≥ threshold | Generate unified diff patch → `fix.patch` |
| `charm-bug` / `test-bug` < threshold | Prose suggestion in report only |
| `flaky` | Retry policy / isolation suggestions in report |
| `infrastructure` | Pre-filled GitHub issue template in report |
| `environment` | Pre-filled GitHub issue template in report |
| `unknown` | Narrative explaining missing signal |

## Decision

Implement `src/glitch/analyze/_remediate.py` with:

1. **`RemediationAction` enum** — `patch`, `suggestion`, `issue_template`, `narrative`
2. **`RemediationPlan` dataclass** — maps verdict labels to actions
3. **`plan_remediation()` function** — decides actions based on threshold
4. **`generate_patch()` function** — calls LLM for unified diff generation
5. **`generate_suggestion()` function** — calls LLM for prose suggestion
6. **Issue template builder** — pre-filled markdown for infrastructure/environment

### Data models

```python
from enum import Enum, auto
from dataclasses import dataclass, field

class RemediationAction(Enum):
    PATCH = auto()
    SUGGESTION = auto()
    ISSUE_TEMPLATE = auto()
    NARRATIVE = auto()

@dataclass
class RemediationEntry:
    test_id: str
    label: str
    confidence: float
    action: RemediationAction
    content: str      # patch diff, suggestion prose, issue template, or narrative text

@dataclass
class RemediationPlan:
    entries: list[RemediationEntry]
    patch_generated: bool   # True if at least one PATCH entry exists
```

### Decision matrix

```python
def plan_remediation(
    verdict: ClassificationVerdict,
    threshold: float,
) -> RemediationPlan:
    entries = []
    for label, confidence in verdict.labels.items():
        if label in ("charm-bug", "test-bug"):
            action = RemediationAction.PATCH if confidence >= threshold else RemediationAction.SUGGESTION
        elif label == "flaky":
            action = RemediationAction.SUGGESTION
        elif label in ("infrastructure", "environment"):
            action = RemediationAction.ISSUE_TEMPLATE
        else:  # unknown
            action = RemediationAction.NARRATIVE
        entries.append(RemediationEntry(...))
    return RemediationPlan(entries=entries, patch_generated=any(...))
```

### Patch generation prompt

For `charm-bug`/`test-bug` ≥ threshold, another LLM call generates a unified diff:

```python
PATCH_PROMPT = """Based on the following failure analysis, generate a unified diff
patch to fix the issue. Output ONLY the patch in unified diff format.

Failure classification: {label} (confidence: {confidence})
Reasoning: {reasoning_trace}

The patch must be applicable with `git apply` and include context lines.
Do NOT include explanations — only the diff."""
```

The raw LLM output is written as-is to `fix.patch`. The spec says only one `fix.patch` per run — if multiple tests need patches, only the highest-confidence one generates a patch; others get suggestions.

### Acceptance criteria

1. `plan_remediation(verdict, 0.8)` produces `PATCH` action when `charm-bug` confidence ≥ 0.8
2. `plan_remediation(verdict, 0.8)` produces `SUGGESTION` when `charm-bug` confidence < 0.8
3. `plan_remediation(verdict, 0.8)` produces `SUGGESTION` for `flaky`, `ISSUE_TEMPLATE` for `infrastructure`, `NARRATIVE` for `unknown`
4. Only one patch is generated per run (highest confidence `charm-bug` or `test-bug` wins)
5. Issue templates include the test_id, reasoning trace, and evidence summary
